Beliefs

👻

Invisible is good.

Invisible is good.

Good design system work is often invisible. If people can focus on the actual problem instead of fighting the tool, that's success — not how polished the file looks.

🤖

AI doesn't replace the work.

AI doesn't replace the work.

AI should take the boring work off your plate: audits, first drafts, repetitive tasks. Some things can be accelerated, not fully replaced. The human touch still matters.

📡

Your spec has a new reader, and it's a machine.

Your spec has a new reader, and it's a machine.

Tokens and docs used to onboard designers. Now they also feed AI tools — which improvise, badly, when the source is ambiguous (like docs for humans!). If your system isn't readable by machines, there's no magic command that will make it build with quality.

🔗

Connecting, not translating.

Connecting, not translating.

Design and code used to tell two different stories. But it's never been easier to make them tell the same one. The fix is not necessarily choosing one over the other, but connecting them from the start.
