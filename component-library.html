Component library

Reusable pieces for the portfolio. Edit once in js/components.js, use everywhere.

site-nav

Global navigation (brand + links + CTA). No attributes. Update links in js/components.js. 

<site-nav></site-nav>

site-footer

Footer with copyright (year is automatic) and tagline. Edit strings in js/components.js. 

<site-footer></site-footer>

project-card

Project/portfolio card. Use attributes: title, description, image, image-alt, href, tags (comma-separated). Add unpublished for coming-soon cards. 

<project-card
  title="IQ Design System"
  description="Building bridges through Design..."
  image="images/cover-ds.png"
  image-alt="Cover image for IQ Design System"
  href="soniq-design-system.html"
  tags="Design System"
></project-card>

<!-- Coming soon (no link) -->

  unpublished
></project-card>

section-heading

Section title. Attributes: text, optional align (center | left). 

<section-heading text="What I do" align="center"></section-heading>
<section-heading text="Some of my work" align="left"></section-heading>

highlight-block

Arrow + highlighted text. Attribute: text.

<highlight-block text="Expert guidance and innovative solutions..."></highlight-block>

contact-link

Contact block (e.g. LinkedIn, Email). Attributes: href, label, sublabel. 

<contact-link
  href="https://www.linkedin.com/in/ericamenin/"
  label="LinkedIn"
  sublabel="in/ericamenin/"
></contact-link>

Article / project page template

For each item under “My work” (and for blog posts), use the same page structure. Copy article-template.html and fill in the components below.

article-nav

Simplified nav for inner pages (logo + “Let’s talk!”). Use on every project and blog page. 

<article-nav></article-nav>

article-header

Title, tags, optional date (for blog), subtitle (inner HTML), featured image. Attributes: title, image, image-alt, tags (comma-separated), date (optional). 

<article-header
  title="Your project or post title"
  image="images/cover.png"
  image-alt="Describe the image"
  tags="Design System, Case Study"
  date="March 15, 2025"
>
  <div>Subtitle with <a href="#" class="link">links</a>.</div>
</article-header>

article-end

Previous + Next tiles and footer (rights + credits) in one block. No space between them, no bottom margin. Attributes: prev-href, prev-title, next-href, next-title (omit any you don’t need; if all omitted, only the footer is shown). 

<article-end
  prev-href="index.html#work"
  prev-title="Back to work"
  next-href="next-project.html"
  next-title="Next Project"
></article-end>

Where to edit

  • js/components.js – Nav links, footer text, and all component logic. Change once, reuse everywhere.
  • article-template.html – Base template for project case studies and blog posts. Copy and rename for each new page.
  • css/emenin.css – Existing styles (layout, typography, colours). Components use these classes.
  • css/components.css – Extra styles for the library page and any component overrides.
